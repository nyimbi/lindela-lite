# Code of Conduct

Lindela Lite is built for public-interest climate, flood, environmental, and service-delivery analysis. The project welcomes contributors from civic technology, humanitarian response, environmental science, public administration, research, and software engineering communities.

## Our Standards

Expected behavior:

- Be respectful and direct.
- Assume good intent while still asking for evidence.
- Keep critique focused on the work, not the person.
- Credit data sources, research, and prior art.
- Respect source licenses and privacy boundaries.
- Avoid publishing information that could endanger people, communities, or response operations.

Unacceptable behavior:

- Harassment, intimidation, insults, or discriminatory language.
- Publishing private, sensitive, operational, or client data.
- Adding credentials, secrets, or non-redistributable datasets.
- Contributing code copied from projects that do not permit reuse.
- Attempting to add GDELT ingestion or proprietary Lindela internals to Lite.
- Using the project to target, surveil, or harm vulnerable people or communities.

## Sensitive Contexts

Climate-conflict analysis can affect real communities. Contributors should avoid overstating model confidence, presenting baseline heuristics as operational certainty, or publishing examples that identify vulnerable individuals or sensitive facilities without a public-interest reason and lawful basis.

## Enforcement

Maintainers may remove comments, issues, pull requests, examples, or contributors that violate this code of conduct. Enforcement actions may include warnings, temporary restrictions, or permanent bans from project spaces.

## Reporting

Report conduct concerns privately to the maintainers or through the repository moderation tools. Include enough detail to evaluate the issue, but do not include sensitive personal data unless necessary.

## Attribution

This code is inspired by common open-source community standards and adapted for climate, humanitarian, and conflict-analysis work.
